# Phase 1.2 実装計画 — 流入経路（お客様の自己申告）分析

> **For agentic workers:** REQUIRED SUB-SKILL: superpowers:subagent-driven-development で1タスクずつ実行。

**Goal:** 予約CSVで取得済みの `howFound`（●ご予約の経緯）と `media`（媒体）を、**既知カテゴリに正規化した `source`** として `history:latest` に追加し、ダッシュボードに「流入経路（自己申告）」カードを出す。広告不要・既存データのみで「流入経路・認知」の問いに直接答える。

**Architecture:** sync が正規化（生の自由記述は**KVに入れない**＝PII防止）→ web `/ingest-history` が `source` を受理 → dashboard が経路別の予約数・売上を集計して既存の棒グラフで表示。3リポジトリとも**加算のみ**。

**Tech Stack:** 既存どおり（TS / vitest。sync=Node、web/dashboard=Cloudflare Worker）。

## Global Constraints
- **生の自由記述（howFound/media）を `history:latest` に入れない**。入れるのは固定カテゴリ文字列 `source` のみ。
- カテゴリ: `Instagram` / `Facebook` / `リピート` / `紹介` / `検索・Web` / `アソビュー` / `その他` / `未回答`（この8種のみ）。
- 判定順序（先勝ち）: Instagram → Facebook → リピート → 紹介 → 検索・Web → アソビュー →（テキストありで不一致）その他。テキスト空 → 未回答。
- 既存テスト（sync 88 / web 240 / dashboard 94）を壊さない。dashboard の `source` は**任意フィールド**（旧データ互換。欠損は「不明」表示）。
- 各リポジトリで typecheck+test を通してからコミット。**push はしない**（ユーザーがGo-Live時にまとめて反映）。

`HistoryRecord` interface に `source: string;` を追加し、`parseAmount` の近くに追加:
```ts
// 「ご予約の経緯」(自由記述)＋「媒体」を既知カテゴリへ丸める。
// 生の自由記述はPII混入の恐れがあるためKVへは出さず、このカテゴリのみを公開する。
// 判定は先勝ち（SNS・リピートを検索より優先）。
export function normalizeSource(howFound?: string, media?: string): string {
  const t = `${howFound ?? ''} ${media ?? ''}`.toLowerCase();
  if (!t.trim()) return '未回答';
  const has = (...words: string[]) => words.some((w) => t.includes(w));
  if (has('インスタ', 'instagram', 'insta')) return 'Instagram';
  if (has('facebook', 'フェイスブック', 'fb')) return 'Facebook';
  if (has('リピート', '以前', '前回', '毎年', '常連', '再訪', '2回目', '二回目')) return 'リピート';
  if (has('紹介', '知人', '友人', '友達', '家族', 'すすめ', '勧め')) return '紹介';
  if (has('検索', 'google', 'グーグル', 'yahoo', 'ヤフー', 'ネット', 'ウェブ', 'web', 'ホームページ', 'サイト')) return '検索・Web';
  if (has('アソビュー', 'asoview', 'あそびゅー')) return 'アソビュー';
  return 'その他';
}
```
`buildHistoryRecords` の map 内に `source: normalizeSource(r.howFound, r.media),` を追加。

```ts
import type { HistoryRecord } from '../types.js';
import { type Period, filterPeriod } from '../period.js';
import type { CourseRow } from './course.js';

// 流入経路（自己申告カテゴリ）別の集計。CourseRow を流用し course にラベルを入れる。
// source 欠損/空（sync 更新前の旧データ）は「不明」に丸める。
export function computeSourceBreakdown(all: HistoryRecord[], period: Period): CourseRow[] {
  const map = new Map<string, CourseRow>();
  for (const r of filterPeriod(all, period)) {
    const label = r.source || '不明';
    const cur = map.get(label) ?? { course: label, bookings: 0, revenue: 0, pax: 0 };
    cur.bookings += 1; cur.revenue += r.amount; cur.pax += r.pax;
    map.set(label, cur);
  }
  return [...map.values()].sort((a, b) => b.revenue - a.revenue);
}
```
